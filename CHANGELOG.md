# Changelog

## 0.13.0 - 2026-07-08
- **Mobile ergonomics:** chapter navigation now lives where the reading stops — a chapter-turn block at the end of every chapter (muted Previous, ink-weight Next with the next chapter's title, "The end — back to library" on the last), sized for thumbs and scaled to the reading font. Icon buttons grow to Apple's 44px floor on touch devices (were 34px). The book-card menu no longer hover-opens *and* click-toggles — a race where the pointer's own hover opened the menu and the click closed it again, which broke every touch tap (synthetic mouseenter precedes click) and was the source of the intermittent `library.spec` CI failure.
- **Loading states grew shapes:** cover-grid shimmer on the library shelf, row shimmer in the chapter list, paragraph shimmer while a chapter's blocks arrive — replacing bare "Loading..." text. Meaningful copy (downloading to device, restoring position) stays as words.
- **Dark-mode images, actually fixed:** dark reader themes print images on a warm paper backing — Standard Ebooks-style engravings are black ink on *transparency* and simply vanished on dark pages; opaque white scans keep the glare dimming from 0.12.0's follow-up. Verified against Tenniel's plates in the illustrated Alice.
- **PWA polish (deferred-list item, closed):** persistent storage requested once signed in (Safari evicts IndexedDB after ~7 idle days otherwise — the silent-library-loss failure mode), an Install entry in the library's ⋯ menu (real prompt on Chromium, Share → Add to Home Screen hint on iOS), and a real icon set (PNG 192/512/maskable + opaque apple-touch-icon; Android install criteria reject SVG-only manifests, iOS ignores manifest icons entirely).
- **Export bakes your metadata in (deferred-list item, closed):** exported EPUBs are rebuilt via spine's writer with the edited title/author/series/description and replaced cover — "Mesmorize by Bob" leaves the building as "M E M O R I Z E by Ro Yu-jin". Best-effort in a worker with graceful fallback to the raw R2 copy; round-trip tested including a 10MB illustrated book with all 44 plates intact. Overrides also replace spine's refined metadata layers (titles[], descriptions[], belongs-to-collection), which otherwise shadow the simple fields.
- **Review hardening:** an 8-angle review of this batch confirmed and fixed 8 findings before release. The big one: replacing a cover left the source cover in the writer spec, so re-exporting any previously-exported file collided with the writer's default cover path (`duplicate_href`) and silently fell back to the raw copy — reproduced live, fixed by dropping the stale resource, regression-tested with a double round-trip. Also fixed: cleared fields (series/description/author) now clear in the export instead of resurrecting from the source (the local row is the identity — absent means cleared); the 44px touch buttons no longer overlap card titles (padding raised in step) or the reader drawer (anchor lowered on coarse pointers); the PWA icons joined the service-worker precache (workbox glob omitted png); the export worker gained the same construction-failure fallback its parse sibling has; the tombstone sweep got real `by_deleted` indexes instead of an unindexed scan that could hit Convex's read ceiling at scale; and the two identity-field lists (re-seed vs export) now cross-reference each other. Deferred knowingly: per-book worker spawns in bulk export, small test-constant duplication.
- **Tombstone compaction (the codebase's one TODO, closed):** a daily Convex cron deletes bookmark/collection tombstones older than 90 days, per the sync design's fixed-horizon rule; README's feature list caught up with reality (it described ~v0.5).

## 0.12.0 - 2026-07-07
- **@abfcode/spine 0.1.1 → 0.8.0.** Seven minor versions of the parser in one jump, adapted end to end: the reader's inline renderer moved to the 0.7.0 model (styling is combinable `emph`/`strong`/`code` flags, so `<strong><em>x</em></strong>` no longer loses a style; legacy-kind blocks still render until they re-parse), the ingest pipeline stopped re-resolving image srcs and link hrefs (spine ≥0.6 pre-resolves them to archive-relative paths — re-joining corrupted every image path, caught by a new real-EPUB corpus test), internal link resolution tries the pre-resolved href directly before falling back to relative resolution for old blocks, and search text extraction no longer injects a space around every styled run (`(sic)`, not `( sic )`). `PARSER_VERSION` now comes from spine's own `VERSION` export, so this bump — and every future one — automatically re-parses the whole library from raw EPUBs on next open, picking up 0.2–0.8's fidelity fixes (nested styling, malformed-XML recovery, entity handling, calibre series fallback) retroactively. Parsing is 5–7× faster upstream; the full 6MB/1,003-section memorize testbook parses in well under a second.
- **EPUB parsing moved off the main thread.** spine 0.4.0 dropped its DOMParser dependency, unblocking the roadmap's deferred Web Worker item: imports and device seeding/re-parse now run in a module worker (image/cover bytes transfer back zero-copy), so a 2,000-chapter webnovel import no longer freezes the UI. Environment failures degrade to a main-thread parse — the worker is an optimization, never a point of failure — and the degraded path logs a marker that the reading e2e asserts is *absent*, proving the worker really runs in a real browser (vitest's browser mode can't host Vite module workers, so unit tests cover the fallback and e2e covers the worker). The worker chunk rides the PWA precache, so offline seeding keeps working. Verifying this also caught a real bug: a parse failure inside the fallback path previously hung the import promise forever.
- **EPUB corpus tests** (`epubCorpus.test.ts`), two tiers: a synthetic image-bearing EPUB always runs (CI included) and pins the src-resolution invariant — every block-referenced image must resolve to extracted bytes (the regression that would have shipped without it); the real `testbooks/` novels additionally run on dev machines (structure pinned: 1,003 sections for memorize, 32 for worm4) but are copyrighted files kept out of the public repo, so those skip in CI.

## 0.11.0 - 2026-07-07
- **NovelUpdates clipper extension** (`extension/`): a one-click companion for the ~100x webnovel-linking flow — right-click → "Copy to Librium" (or the toolbar icon) on any NU series page copies the page HTML *and* the cover image as one JSON clipboard payload; pasting into the Edit-details dialog fills the source URL, opens the candidate diff, and stages the cover in one go. Getting the cover took three attempts, live-tested against real Cloudflare: extension background fetch is 403'd even with `credentials:"include"` (the clearance cookie is partition-bound to the tab), page-context fetch is CORS-sealed (CDN sends no `Access-Control-Allow-Origin`), so the shipped path **screenshots the rendered page and crops the cover** (`captureVisibleTab` + `OffscreenCanvas`) — no network request, nothing for Cloudflare or CORS to judge. That API demands the `activeTab` grant, which browsers hand out only for clicks on extension UI — hence context-menu/toolbar triggers and no injected button (a v0.1.x floating button could never capture; replaced by a status toast). Both doomed fetch paths were deleted after the live verdicts (git history keeps them should NU ever relax), so a successful run's console stays clean. Cross-browser MV3 (Chrome service worker + Firefox event page via dual background keys; `browser_specific_settings.gecko` set), host access scoped to novelupdates.com only, no server contact — clipboard out only. Versioned payload contract (`librium: 1`, diagnostic `coverError` on failure) parsed by `parseLibriumPayload` with per-field validation; unrecognized pastes fall through to normal text behavior. The captured payload also became a test fixture: `parseNovelUpdatesHtml` is now verified against real live NU markup (`web/src/test/fixtures/nu-memorize.html`), retiring the "selectors unverified" caveat.
- **Covers past Cloudflare:** NovelUpdates fronts its image CDN with the same Cloudflare wall as its pages (verified: `cdn.novelupdates.com` 403s every server-side fetch regardless of User-Agent), so the cover proxy could never fetch an NU candidate's cover and Save died with a bare 403 *after* the text fields had already committed. Three fixes: fetched-cover downloads now try the **user's browser first** (real browser traffic passes Cloudflare, and CORS-friendly hosts like Open Library / Google Books allow reading the bytes) with the server proxy as fallback; a cover failure is **non-fatal** — details stay saved and the dialog explains the way forward instead of a bare error; and you can now **paste an image (Ctrl+V)** anywhere in the Edit-details dialog to set the cover — right-click → Copy image on the NU page, paste, Save. No download-to-disk required.
- **Review hardening (clipper + cover pipeline):** an 8-angle review of the extension and dialog changes confirmed and fixed 6 findings. Highlights: applying an NU candidate's diff no longer clobbers a staged file cover (pasted/extension-captured) with the candidate's url cover — the cover row is checked by default and NU CDN urls are guaranteed to 403 at Save, so `applyFetched` now never downgrades file→url (falsifiable regression test included); the browser-first cover download enforces the same 4MB cap as the server proxy (an oversized direct hit previously uploaded uncapped); the extension's context menu survives Firefox restarts (removeAll+create on `onStartup`, not just `onInstalled`); a clipboard image item that yields no File no longer swallows a co-present payload paste; a truncated clipper payload is caught by prefix instead of dumping ~200KB of JSON into the focused field; and a payload whose cover fails to decode now says so instead of silently proceeding. Cleanups: the extension's two dead fetch paths (~120 lines, unreachable since every real NU cover lives on the blocked CDN) are deleted along with the now-unneeded `cdn.novelupdates.com` host permission, and the dialog's three copy-pasted pending-cover constructions collapsed into one `setCoverFromBlob`.
- **"Recent" sort no longer moves when you set a status.** Marking a book (or bulk-marking) bumps the userBooks sync clock — which the cross-device status merge legitimately needs — but that same clock also drove the default Recent shelf, so shelving an old book as Abandoned vaulted it to the top past what you were actually reading. Reading recency now lives in its own `lastActivityAt` field, written only by opening/reading a book (and at import), so status edits leave the shelf order alone. Never-opened books marked from the shelf sort last in Recent, as they should. New falsifiable e2e (`recentSort.spec.ts`) that fails against the old behavior — writing it also exposed that its first draft passed vacuously by racing the async status push, now synchronized.

## 0.10.0 - 2026-07-07
- **Edit details:** every book gets an editor (card menu → "Edit details…") — title, author, series/index, description, language, and a source-page URL; replace the cover from a local image. Saves only changed fields (empty = clear); server-authoritative and online-only, mirrored into the local shelf row so offline views agree. Replaced covers propagate to other devices via a `coverUpdatedAt` stamp (the R2 key never changes, so staleness is version-checked, on the server's clock).
- **Metadata fetch:** a "Fetch metadata" panel searches Open Library (no key) and Google Books (set `GOOGLE_BOOKS_API_KEY` on the Convex deployment to enable) — ISBN-first when the EPUB carried one, title/author otherwise. Candidates show covers and sources; picking one opens a per-field diff (current vs proposed, checkboxes) that merges into the form — **nothing touches the server until Save**. Fetched covers download through a server proxy (image hosts don't send CORS) with https-only/private-host/size guards, then ride the normal R2 + local-blob path.
- **NovelUpdates linking:** paste a series URL into the source-page field — "Open source page" everywhere it's shown, and "Fetch from linked page" pulls title/author/description/genres/cover from that exact page (no search matching). NU's Cloudflare blocks server fetches routinely, so the flow degrades to paste-the-page-HTML, parsed client-side into the same diff preview. The page-fetch action is allowlisted to novelupdates.com only.
- **Re-seed identity preservation:** re-downloading a book's content (device seeding, stale-parser refresh) no longer resurrects the EPUB's embedded metadata or cover over user edits — re-parse refreshes content, never identity. Covered by e2e (edit → remove download → re-download → edit intact).
- Reconcile now also mirrors drifted server metadata into local rows and invalidates stale local cover blobs; the R2 upload helper is shared (`uploadBookAsset`) between import and the editor.
- **Review hardening:** a 6-angle adversarial review (both 0.9.0 and 0.10.0 code) confirmed and fixed 18 findings across sync, security, and UI. Highlights: a collection membership un-checked during its create/add round-trip is no longer stranded alive on the server (dirty-flag stomp past a tombstone); status/collection merge writes now re-read the live row in a transaction so a concurrent edit can't be clobbered back to `dirty:0`; the metadata fetch actions re-validate **every redirect hop** (SSRF-via-redirect close), block IPv6 loopback/ULA/link-local/mapped hosts, and stream-abort oversized bodies instead of trusting `Content-Length`; `attachFiles` binds the R2 key to the book's own prefix (closes a cross-user object-read via a forged key); the Edit-details dialog diffs against its open-time snapshot (not the live doc) so a concurrent edit from another device isn't reverted on Save; "Select all" selects only the visible/filtered shelf; the delete-collection confirm no longer renders under the Manage dialog; a stale reconcile run can no longer demote a freshly-seeded book to content-absent (cancellation guard + transactional insert); the import path stamps `coverVersion` so a freshly imported cover isn't dropped and re-downloaded; and Escape no longer both closes a dialog and drops the multi-selection. Low-severity sync edge cases (LWW loser convergence, offline account-switch) and UX papercuts (paste-to-parse, search using live form terms) also fixed. A dedicated data-integrity pass over the reconcile/seed/cover machinery found no persistent-corruption defect. Two known-lows are deferred: marking a status bumps the "Recent" sort (needs a separate recency field), and cover *replacement* isn't atomic across the R2 upload + `attachFiles` stamp (an interrupted replace can show a stale cover on other devices until a retry — inherent to the fixed cover key).

## 0.9.0 - 2026-07-07
- **Reading status:** every book has a status — Reading / Want to read / Finished / Abandoned — set from the card menu (re-picking clears back to Automatic) or in bulk via "Mark as" over a selection. Unset books derive their status from progress (untouched → Want to read, started → Reading, ≥99% → Finished). Local-first with LWW sync on its own clock (`statusEditedAt`, disjoint from progress), so status edits work offline and never clobber or get clobbered by progress writes.
- **Collections:** user-named, many-to-many book groups (a book can live in several) — "Add to collection…" from the card menu or over a bulk selection opens a picker with inline create; a Manage dialog renames/deletes. Fully offline-capable on the bookmark tombstone pattern (idempotent clientKey creates, deletedAt tombstones), with one new rule: memberships reference collections by client key, and pushes send collections first so books added to an offline-created collection sync correctly on reconnect.
- **Series shelves:** a new "Series" sort groups the library by the EPUB's series metadata, volumes ordered numeric-aware ("Vol 2" before "Vol 10", fractional indexes like "1.5" in place); standalone books trail in one group. Works offline (series fields mirrored into the local shelf rows).
- **Shelf filters:** a filter row under the header — status tabs, an "On this device" toggle (downloaded-only), and a Collection dropdown — all composable with search and sort, persisted across visits (`library:filters`).
- **Sync hardening:** push passes now serialize on a promise queue and re-read dirty rows from IndexedDB — a change landing while a push was in flight could previously be dropped, stranding unsynced edits until the next unrelated write.
- **Library toolbar extracted** (`LibraryToolbar`) — LibraryView had grown past 1,100 lines; the header/filter chrome is now a presentational component.
- New e2e journey (`library.spec.ts`): status via menu → shelf tabs → collection create/file/filter → reload persistence; browser tests for the collection-sync push ordering, picker dialog, and every new filter.

## 0.4.0 - 2026-07-04
- **Deleted-book handling:** a book deleted from another device no longer error-pages a reader that has it open — the reader purges the local copy and returns to the library (`books.getBook` null signal; reader-path queries return empty instead of throwing).
- **Local-first library + bookmarks (ROADMAP Phase 4, complete):** the library shelf renders from IndexedDB when offline (covers cached as local blobs; progress badges derived from local records) and reconciles against the authoritative server list when online — books deleted on another device are purged locally, and books imported elsewhere appear on every device's shelf. Bookmarks are local-first with tombstone sync: create/delete work offline, offline creates push idempotently on reconnect (client keys), and deletes propagate as server tombstones instead of being resurrected by other devices.
- **Local-first progress + LWW sync (ROADMAP Phase 4, slice 1):** reading progress is written to IndexedDB first (instant, offline-durable) and synced to Convex with last-write-wins: dirty local edits push on edit and on reconnect; the server rejects pushes older than what it holds (`progressEditedAt`), so a stale queued write can never clobber newer progress; pulls adopt remote state only when its server timestamp is newer than the last merged state and never over unpushed local edits. The reader restores from the merged view — offline reopen resumes at the exact local position; a newer remote position corrects the view only within moments of opening and never after the reader has navigated (no mid-session yanks).
- **PWA shell (ROADMAP Phase 3):** `vite-plugin-pwa` service worker precaches the app shell (JS/CSS/fonts/icon) with SPA navigation fallback — production builds boot fully offline (content already comes from IndexedDB). Installable: web app manifest + icon. Applies to `pnpm build` output only; the dev server does not emit the service worker.
- **Local-first read path (ROADMAP Phase 2):** the reader now reads from the device. Import writes the parsed book (sections + blocks + images + cover) to IndexedDB (Dexie) before ingest — readable immediately; chapter turns, TOC, and images are served from IndexedDB with no network I/O. Books imported on another device fall back to Convex and cache-fill locally (section metadata, blocks, and images) so subsequent reads are local. Deleting a book purges the local copy. `RequireAuth` gains offline grace: a previously signed-in device renders local content when offline. Progress/bookmarks still sync via Convex (durable local progress lands with the Phase 4 sync layer).
- **Static SPA (ROADMAP Phase 1):** removed TanStack Start and the nitro SSR server; the app is now a plain Vite + TanStack Router single-page app (`index.html` + `src/main.tsx` entry, route code-splitting via `@tanstack/router-plugin`). `vite build` emits a fully static `dist/` deployable to any static host. Auth is unaffected (Better Auth is served by Convex HTTP actions, not the removed Node server). Drops the `nitro` nightly pin and `@tanstack/react-start`/`react-router-ssr-query` dependencies.
